Wed Jun 10 2020

Some languages like Go cannot convert directly from number to date string:

package main
import "time"
func main() {
   var n int64 = 365 * 24 * 60 * 60
   var o = time.Unix(n, 0)
   var s = o.String()
   println(s)
}

Same with JavaScript:

let n = 365 * 24 * 60 * 60;
let o = new Date(n * 1000);
let s = o.toString();
console.log(s);

PHP can:

<?php
$n = 365 * 24 * 60 * 60;
$s = strftime('%F', $n);
var_dump($s);

and PHP can also convert from object:

<?php
$n = 365 * 24 * 60 * 60;
$o = new DateTime;
$o->setTimestamp($n);
$s = $o->format('Y-m-d');
var_dump($s);

we could approach like this:

  • get date number
  • get date object
  • get date string

but those pages could be too large, as could be multiple input and output options. Better would be this:

  • number to object
  • number to string
  • object to number
  • object to string
  • string to number
  • string to object

So in this case, we can split like this:

  • number to string
  • number to object
