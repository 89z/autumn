Condition usage

If statements

PHP accepts any type with conditions:

<?php
$n1 = 10;
# boolean
if ($n1 == 0) { # false
   $n2 = 20;
} else {
   $n2 = $n1;
}
# number
if ($n1) { # true
   $n3 = $n1;
} else {
   $n3 = 20;
}
# print
var_dump($n2, $n3); # 10 10

Go accepts booleans with conditions:

package main
func main() {
   var n1 = 10
   var n2 int
   if n1 == 0 { // false
      n2 = 20
   } else {
      n2 = n1
   }
   println(n2) // 10
}

but not other types:

package main
func main() {
   var n1 = 10
   var n2 int
   if n1 { // non-bool n1 (type int) used as if condition
      n2 = n1
   } else {
      n2 = 20
   }
   println(n2)
}

To deal with this, we should simply only use booleans when a condition is expected.

Ternary operations

PHP offers this:

<?php
$n1 = 10;
$n1 = $n1 == 0 ? 20 : $n1;
echo "$n2\n"; # 10

https://php.net/operators.comparison#language.operators.comparison.ternary

Go does not offer this:

The reason ?: is absent from Go is that the language’s designers had seen the operation used too often to create impenetrably complex expressions. The if-else form, although longer, is unquestionably clearer. A language needs only one conditional control flow construct.

https://golang.org/doc/faq#Does_Go_have_a_ternary_form

We can continue to use ternary opeations, as long as the condition is a boolean. For example with Python:

n1 = 10
n2 = 20 if n1 == 0 else n1
print(n2) # 10

or JavaScript:

let n1 = 10;
let n2 = n1 != 0 ? n1 : 20;
console.log(n2); // 10

Binary operators

The first argument is a condition. As such, per the previous decision, it must be a boolean. For example with JavaScript:

let n1 = 10;
let n2 = n1 != 0 || 20;
console.log(n2); // true

or PHP:

<?php
$n1 = 10;
$n2 = $n1 != 0 ?: 20;
var_export($n2); # true

This isnt really useful if we want to return non boolean values. Further, some languages like PHP coerce the result to boolean:

<?php
$n1 = 10;
$n2 = $n1 || 20;
var_export($n2); # true

or worse, only fail on null:

<?php
$b1 = false;
$b2 = $b1 ?? true;
var_export($b2); # false

https://php.net/operators.comparison#language.operators.comparison.coalesce

In order to have two values that can be returned, we would need to use an if statement or similar. However implementing an if statement means that the second argument is also a condition, and must be a boolean as well. Better would be to use boolean type with all arguments. For example with JavaScript:

let n1 = 10;
let n2;
if (n1 > 40 || n1 < 20) { // true
   n2 = n1;
} else {
   n2 = 30
}
console.log(n2); // 10

Further, some languages like Go only accept boolean values:

package main
func main() {
   n1 := 10
   n2 := n1 || 20 // (operator || not defined on int)
   println(n2)
}

https://golang.org/ref/spec#Logical_operators

Constants

It should be noted that certain languages like Go do not support conditional constants. These fail with JavaScript:

if (true) {
   const n1 = 10;
}
console.log(n1); // ReferenceError: n1 is not defined

switch (true) {
case true:
   const n1 = 10;
}
console.log(n1); // ReferenceError: n1 is not defined

this works:

const n1 = {true: 10}.true;
console.log(n1);
